# Handoff — Studio Recorder

Fecha: 2026-05-17

## Objetivo del producto
Studio Recorder es un estudio web de grabacion y edicion tipo streaming, libre y sin registro, pensado para grabar:
- pantalla
- camara del usuario
- microfono
- audio del sistema cuando el navegador lo permite

Y luego editar ese material en el mismo flujo, sin backend y sin subir archivos a terceros.

## Flujo funcional actual
1. Entrar a `/studio`
2. Pulsar `Detectar camara y microfono`
3. Autorizar permisos del navegador
4. Elegir pantalla
5. Seleccionar camara y microfono
   - opcionalmente elegir fuente de audio: mic, sistema o ambos
6. Grabar
7. Al detener, la grabacion:
    - se guarda en biblioteca local
    - queda disponible para reproducir, descargar o abrir manualmente en el editor
8. En el editor se puede:
   - importar medios
   - agregar texto
   - recortar
   - mover clips
   - duplicar
   - cortar
   - ajustar volumen/mute
    - mover clips entre pistas compatibles
    - colapsar pistas
    - agregar o quitar canales vacios
    - ajustar volumen y paneo por canal, y volumen fino por clip
    - usar snapping en timeline
   - mover el playhead con click, drag o scrub en la timeline
   - introducir tiempos manuales para previsualizar partes concretas
   - guardar proyecto
   - exportar WebM o MP4
   - mover y escalar elementos visuales sobre el preview

## Riesgos o puntos a vigilar
- El audio de sistema depende del navegador/OS/fuente compartida
- El modo `solo sistema` requiere que la fuente compartida entregue track de audio; si no, la UI bloquea la grabacion y pide cambiar fuente o modo
- El editor ya es funcional, pero el archivo `StudioEditor.tsx` sigue siendo grande y conviene partirlo
- El export MP4 local puede ser pesado en CPU/memoria en equipos modestos
- La generacion de waveforms usa `AudioContext.decodeAudioData()` y puede ser costosa con archivos muy largos
- La manipulacion directa en preview existe, pero aun puede mejorar con mas handles y restricciones visuales
- El layout vertical del editor depende de alturas acotadas entre preview y timeline; futuros ajustes deben validarse visualmente ademas de `check` y `build`
